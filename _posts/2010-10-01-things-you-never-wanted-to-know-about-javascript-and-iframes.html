Sizing and Resizing

If an iframe is in a window that is smaller than its needs for minimum height and width, and you haven't done anything to suppress them, you'll get scrollbars. The interesting thing is when we use a little bit of JavaScript to get the sizing:

In this example we'll just assume that iframe is the iframe element we want to be operating on.

What's interesting about the above example is that despite the parent window being a smaller size than the required area of the iframe, the sizing values we'll get back will be the actual height and width of that the contained page requires. So, if you're using something like a 960px grid then it is very likely that the value will be '960px'.

Even more interesting is what will happen if we increase the window to be large enough to properly display the entire iframe and its contents. If we were to run that JavaScript again, the values returned will be the sizing of the parent as well not simply the minimum required space for the iframe.

While interesting, this information is really very useful if you're trying to match the size of other elements on a page, or do some resizing. In my case I was on a quest to match the size of an iframe with a canvas element in order to add an overlay.
